# Layer 0–8 Purpose/Obligation/Verification Consistency Audit

> **Historical Reference Notice:** Dashboard dataset citations (e.g., `analytics/dashboard/…`) refer to private Governance Control Tower™ assets retained for archival transparency and are not part of the open-core repository.

## Scope & Method
- Reviewed autogenerated control tables for Layers 0–8 to confirm each contains a Purpose statement, explicit obligations (`shall/should`), a named verification artefact, and a mapped AEIP schema reference.
- Cross-referenced dashboard data payloads (`layers.json`, `update_plans.json`, `artifacts.json`) to ensure obligations in narrative content align with evidence hooks and artefact catalogues.
- Validated AEIP schema files under `schemas/` for naming accuracy, layer alignment, and field coverage. Highlighted discrepancies where schema metadata conflicts with layer usage.

## Additional Observations
- Artifact catalog entries reference layer-specific schema paths, reinforcing the Purpose → Obligation → Evidence mapping for most layers.【F:analytics/dashboard/src/data/artifacts.json†L1-L61】
- AEIP receipts seeded in the dashboard span civic charter, reasoning, instruction, and publication layers; no federation receipt is present yet, mirroring the schema gap.【F:analytics/dashboard/src/data/aeip/index.ts†L1-L17】
- Obligations across tables consistently use `\shall{}`/`\should{}` syntax with accompanying AEIP schema rows, so there are no orphaned SHALL directives within the autogenerated tables themselves.【F:source/chapters/generated/layer0-control-table.tex†L5-L28】【F:source/chapters/generated/layer5-control-table.tex†L5-L29】

## Remediation Recommendations
1. **Realign Layer 1 and Layer 2 Schemas**: Point L1 obligations to the Trusted Execution & Compute Log (`tecl`) schema and retarget L2 to the Curated Custody/Consent Manifest (`ccm`). This restores layer-to-schema parity and eliminates the current `L2` enumeration conflict.【F:source/chapters/generated/layer1-control-table.tex†L15-L23】【F:schemas/aeip/ccm-schema.json†L1-L112】【F:schemas/aeip/tecl-schema.json†L1-L129】
2. **Introduce Data Stewardship AEIP Schema**: Author a dedicated lineage/consent register schema (e.g., `lineage-ledger.jsonld`) to replace the Decision Rationale Record reference in L2 tables.【F:source/chapters/generated/layer2-control-table.tex†L15-L23】【F:schemas/drr-schema.yaml†L1-L88】
3. **Create Federation Sync Schema**: Publish an AEIP schema for `federationSync` events so L8 obligations have a matching verification artefact; update control tables and data payloads accordingly.【F:source/chapters/generated/layer8-control-table.tex†L15-L23】【F:analytics/dashboard/src/data/update_plans.json†L56-L63】【7dcbd7†L1-L2】
4. **Backfill AEIP Receipts**: Extend `analytics/dashboard/src/data/aeip/` with federation and data stewardship receipts once schemas exist to keep the dashboard evidence catalogue complete.【F:analytics/dashboard/src/data/aeip/index.ts†L1-L17】
